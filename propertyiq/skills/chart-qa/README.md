# chart-qa — Chart Visual Audit Skill

Post-deploy visual validation of charts against `propiq-docs/charts/spec.yaml`.

## When PM uses this

After a merge to `main` on any chart-stack repo (`propiq-charts-api`, `propiq-charts-img`, `propiq-reports-web`, `propiq-charts-maps`), PM runs a chart-validation pass. This skill provides the evaluation flow, spec-fetch mechanism, and output template.

## Files

| File | Purpose | Editable? |
|------|---------|-----------|
| `SKILL.md` | Evaluation flow and decision tree | Yes (via PR) |
| `fetch_spec.sh` | Vendors `spec.yaml` + `tokens.yaml` from propiq-docs at audit time | Yes (via PR) |
| `audit-finding-template.md` | Structured Issue body for chart-audit findings | Yes (via PR) |
| `piq-style-quickref.md` | Human-scannable summary of all spec rules | **No** — auto-generated by `scripts/regen_quickref.py` |

## Important

- `piq-style-quickref.md` is regenerated from `spec.yaml` on every PR via CI. Do not hand-edit it.
- This skill was migrated from the Mac mini's Mini-workspace local path. **Do not duplicate it back to the Mini** — the repo copy is authoritative. The Mac mini picks it up via the existing P0 `git pull` step in HEARTBEAT.md.
